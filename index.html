Why Creating Static Websites Is Cool

Static Pages? Is this 1993?

No doubt you've heard all about blogging engines like WordPress and Blogger, and how they make publishing content easier. And it's true -- the industry has made publishing available to many different people.

But there are a few drawbacks to using database driven websites as well:

  • Having to maintain a database
  • Protecting your sites against hackers/code vulnerabilities
  • Slow website performance, especially with shared hosting

Luckily, static publishing hasn't gone away. In fact, it's making a resurgence!

Creating Static Websites with Github

What's Github?

Github is a repository based web-hosting service. It allows users to share and distribute all kinds of content. Based on Git, a distributed revision control system, Github provides a web-based graphical interface, desktop clients and mobile integration.

So What?

Say you're working on a project - you're comparing library lending data with several libraries across the country. With Github, you can share your raw code, keep track of the latest revisions, and even allow people to build other projects based on your data. Social coding is the future.

Get to the Point, Cecily

Alright, alright.

I'm going to show you how to build a simple static website using Github. There are some very simple steps.

  • Create a Github account
  • Create a repository called username.github.io
  • Go to your settings page and click Automatic Page Generator to create your website
  • Go to Layouts to select a layout for your page
  • Click Publish Page to..er..publish (it may take up to 10 minutes for your page to show)

Once your site has generated you can use the Github client (or the command line if you're hardcore -- and if you're in DSV, you're definitely hardcore) to sync your pages locally and on Github's servers.
